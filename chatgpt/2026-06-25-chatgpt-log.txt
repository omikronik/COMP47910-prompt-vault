Coming from **React/Next.js** and some **ASP.NET Razor**, the biggest mindset shift is:

> Thymeleaf is not a frontend application. It's server-side HTML templating.

You are not building components that run in the browser. You're building HTML pages that Spring renders on the server and sends to the client.

For an assignment like PromptVault, I would keep it extremely simple.

---

# Think MVC, not SPA

In React you think:

```text
Page
 ├─ Components
 ├─ State
 ├─ API calls
 └─ Routing
```

In Spring + Thymeleaf think:

```text
Controller
 ├─ Gets data
 ├─ Returns View
 │
View (Thymeleaf)
 ├─ Displays data
 └─ Posts forms

Service
 └─ Business logic

Repository
 └─ Database
```

Example:

```java
@GetMapping("/prompts")
public String listPrompts(Model model) {

    var prompts = promptService.getPromptsForUser();

    model.addAttribute("prompts", prompts);

    return "prompt/list";
}
```

Thymeleaf:

```html
<tr th:each="prompt : ${prompts}">
    <td th:text="${prompt.title}"></td>
</tr>
```

No API.
No fetch.
No useEffect.

# Equivalent to React Components

Use Thymeleaf fragments.

React:

```jsx
<Navbar />
```

Thymeleaf:

```html
<div th:replace="~{fragments/navbar :: navbar}"></div>
```

---

## navbar.html

```html
<nav th:fragment="navbar">
    ...
</nav>
```

---

# Use a Layout

Instead of duplicating HTML.

Create:

```text
templates/fragments/layout.html
```

Example:

```html
<!DOCTYPE html>
<html>
<head>
    <title th:text="${title}">PromptVault</title>
    <link rel="stylesheet" href="/css/app.css">
</head>

<body>

<div th:replace="~{fragments/navbar :: navbar}"></div>

<main>
    <div th:insert="${content}"></div>
</main>

</body>
</html>
```

Although for a university assignment, honestly, even just:

```html
<header>
<nav>...</nav>
</header>

<main>
...
</main>
```

copied into each page is acceptable if time is limited.

# Forms Are The Main Interaction Model

React:

```jsx
onSubmit={() => savePrompt()}
```

Thymeleaf:

```html
<form th:action="@{/prompts/create}"
      method="post">

    <input
        type="text"
        name="title">

    <button type="submit">
        Save
    </button>

</form>
```

Controller:

```java
@PostMapping("/create")
public String create(CreatePromptRequest request) {

    promptService.create(request);

    return "redirect:/prompts";
}
```

The controller receives the form directly.

# DTO Convention

Coming from .NET, think ViewModels.

Don't bind entities directly.

```text
dto/
│
├── auth/
│   ├── LoginRequest
│   └── RegisterRequest
│
├── prompt/
│   ├── CreatePromptRequest
│   ├── UpdatePromptRequest
│   └── PromptViewModel
│
└── admin/
```

Controller:

```java
@PostMapping
public String create(
    @Valid CreatePromptRequest request
)
```

Very similar to ASP.NET MVC.

# For Tables

Admin users page:

```html
    <th>Username</th>
    <th>Email</th>

<tr th:each="user : ${users}">
    <td th:text="${user.username}"></td>
    <td th:text="${user.email}"></td>
    <td th:text="${user.active}"></td>
</tr>

</table>
```

This replaces React's:

```jsx
users.map(...)
```

# Flash Messages

You'll use these a lot.

Controller:

```java
redirectAttributes.addFlashAttribute(
    "success",
    "Prompt created successfully"
);

return "redirect:/prompts";
```

View:

```html
<div
    class="alert"
    th:if="${success}"
    th:text="${success}">
</div>
```

Very common Spring MVC pattern.
