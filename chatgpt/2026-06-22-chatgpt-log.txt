how do i start a maven cli app

$GPT
To start a basic Java CLI app with Apache Maven, you usually:

1. Generate a Maven project
2. Add a `main` method
3. Build and run it

## 1. Generate a Maven project

If Maven is installed:

```bash
mvn archetype:generate \
  -DgroupId=com.example \
  -DartifactId=my-cli-app \
  -DarchetypeArtifactId=maven-archetype-quickstart \
  -DinteractiveMode=false
```

This creates:

```text
my-cli-app/
├── pom.xml
└── src/
    ├── main/java/com/example/App.java
    └── test/java/com/example/AppTest.java
```

Move into it:

```bash
cd my-cli-app
```

---

## 2. Edit the main class

Open:

```text
src/main/java/com/example/App.java
```

Replace it with:

```java
package com.example;

public class App {
    public static void main(String[] args) {
        System.out.println("Hello from Maven CLI app!");

        for (String arg : args) {
            System.out.println("Arg: " + arg);
        }
    }
}
```

---

## 3. Build the app

```bash
mvn package
```

This produces:

```text
target/my-cli-app-1.0-SNAPSHOT.jar
```

---

## 4. Run it

```bash
java -cp target/my-cli-app-1.0-SNAPSHOT.jar com.example.App hello world
```

Output:

```text
Hello from Maven CLI app!
Arg: hello
Arg: world
```

---

## Optional: make the JAR executable

Add this inside `<build>` in `pom.xml`:

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-jar-plugin</artifactId>
            <version>3.4.2</version>
            <configuration>
                <archive>
                    <manifest>
                        <mainClass>com.example.App</mainClass>
                    </manifest>
                </archive>
            </configuration>
        </plugin>
    </plugins>
</build>
```

Then rebuild:

```bash
mvn package
```

Now you can run:

```bash
java -jar target/my-cli-app-1.0-SNAPSHOT.jar
```

---

how do i add maven to nix using nix-shell -p

$GPT
You can temporarily enter a shell with Apache Maven using:

```bash
nix-shell -p maven
```

That gives you Maven plus a compatible JDK.

Check it:

```bash
mvn -v
```

---

If you also want a specific Java version:

```bash
nix-shell -p maven jdk17
```

or:

```bash
nix-shell -p maven jdk21
```

---

For a reusable project shell, create `shell.nix`:

```nix
with import <nixpkgs> {};

mkShell {
  buildInputs = [
    maven
    jdk21
  ];
}
```

Then enter it with:

```bash
nix-shell
```

---

If you're using flakes, use `nix develop` instead:

```nix
{
  description = "Maven CLI app";

  inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

  outputs = { self, nixpkgs }:
    let
      system = "x86_64-linux";
      pkgs = import nixpkgs { inherit system; };
    in {
      devShells.${system}.default = pkgs.mkShell {
        buildInputs = [
          pkgs.maven
          pkgs.jdk21
        ];
      };
    };
}
```

Then:

```bash
nix develop
```

Useful docs:

* [NixOS Search (packages)](https://search.nixos.org/packages?utm_source=chatgpt.com)
* [Nixpkgs Manual](https://nixos.org/manual/nixpkgs/stable/?utm_source=chatgpt.com)

can i make it so that i can run this with a single line for easier debugging

$GPT
Yes. The easiest way is to use the Apache Maven exec plugin so you can run:

```bash id="hifw7h"
mvn exec:java
```

or with args:

```bash id="d5fygv"
mvn exec:java -Dexec.args="hello world"
```

Add this to your `pom.xml`:

```xml id="n1ls6n"
<build>
    <plugins>
        <plugin>
            <groupId>org.codehaus.mojo</groupId>
            <artifactId>exec-maven-plugin</artifactId>
            <version>3.5.0</version>
            <configuration>
                <mainClass>com.example.App</mainClass>
            </configuration>
        </plugin>
    </plugins>
</build>
```

Then just:

```bash id="zly3jg"
mvn compile exec:java
```

That compiles and runs in one command.

---

For debugging with the JVM debugger:

```bash id="sqv1kk"
mvn exec:java \
  -Dexec.jvmArgs="-agentlib:jdwp=transport=dt_socket,server=y,suspend=y,address=*:5005"
```

Then attach your IDE to port `5005`.

---

You can also make a tiny shell alias in your `shell.nix`:

```nix id="q4st9w"
shellHook = ''
  alias run="mvn compile exec:java"
'';
```

Then inside the nix shell:

```bash id="9xq7jr"
run
```

or:

```bash id="x2t6b7"
run -Dexec.args="test"
```

You haven't told Apache Maven which class contains `main()` yet.

Add this to your `pom.xml`:

```xml id="5b7xqg"
<build>
    <plugins>
        <plugin>
            <groupId>org.codehaus.mojo</groupId>
            <artifactId>exec-maven-plugin</artifactId>
            <version>3.6.3</version>
            <configuration>
                <mainClass>com.example.App</mainClass>
            </configuration>
        </plugin>
    </plugins>
</build>
```

Replace `com.example.App` with your actual package/class name.

For example, if your file is:

```text id="o4q35m"
src/main/java/com/example/App.java
```

and contains:

```java id="a9ek3f"
package com.example;

public class App {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}
```

then `com.example.App` is correct.

```bash id="3b3tv9"
mvn compile exec:java
```

```bash id="vmpv0k"
mvn exec:java -Dexec.mainClass="com.example.App"
```
