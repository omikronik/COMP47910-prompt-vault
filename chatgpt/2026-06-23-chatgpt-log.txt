---

### 2. Batch save

Instead of saving one at a time:

```java
for (User user : users) {
    if (userRepository.findByEmail(user.getEmail()).isEmpty()) {
        userRepository.save(user);
    }
}
```

You could collect missing users and do:

```java
List<User> usersToSave = users.stream()
    .filter(user -> userRepository.findByEmail(user.getEmail()).isEmpty())
    .toList();

userRepository.saveAll(usersToSave);
```

Not a huge deal for 3 users, but it's a common pattern.

---

### 3. Consider checking once

If this seeder only ever runs on startup, an even simpler pattern is:

```java
if (userRepository.count() == 0) {
    userRepository.saveAll(users);
}
```

The advantage is simplicity.

The disadvantage is if you manually delete one user later, it won't get recreated.

For seed data, I actually prefer your email-based existence check because it's more resilient.

What you're describing is very standard in the Java/Spring world.

The typical stack is:

* **SLF4J** → logging API (`logger.info()`)
* **Logback** → logging implementation (comes with Spring Boot)
* Configuration in `logback-spring.xml`

You do **not** need Log4j2 or another library unless you specifically want it. Spring Boot ships with Logback and it's excellent.

---

# 1. Use Spring Boot's built-in logging

You likely already have it via:

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

which pulls in:

```text
SLF4J
Logback
```

automatically.

No additional Maven dependencies needed.

---

# 2. Create logback-spring.xml

Create:

```text
src/main/resources/logback-spring.xml
```

---

# 3. Configure daily rolling logs

```xml
<?xml version="1.0" encoding="UTF-8"?>

<configuration>

    <property name="LOG_PATH" value="logs"/>

    <appender name="FILE"
              class="ch.qos.logback.core.rolling.RollingFileAppender">

        <file>${LOG_PATH}/promptvault.log</file>

        <rollingPolicy
            class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">

            <fileNamePattern>
                ${LOG_PATH}/%d{yyyy-MM-dd}-promptvault.log
            </fileNamePattern>

            <maxHistory>30</maxHistory>

        </rollingPolicy>

        <encoder>
            <pattern>
                %d{yyyy-MM-dd HH:mm:ss.SSS}
                [%thread]
                %-5level
                %logger{36}
                %M
                - %msg%n
            </pattern>
        </encoder>

    </appender>

    <root level="INFO">
        <appender-ref ref="FILE"/>
    </root>

</configuration>
```
