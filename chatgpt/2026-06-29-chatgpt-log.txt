This is a common point of confusion with Spring MVC. The URL and the template location are independent.

If your controller is

```java
@Controller
@RequestMapping("/dashboard")
public class DashboardController {

    @GetMapping
    public String dashboard() {
        return "dashboard";
    }

}
```

then Spring will look for

```
templates/dashboard.html
```

because you're returning `"dashboard"`.

If your file is actually

```
templates/
    dashboard/
        index.html
```

then you need to return

```java
return "dashboard/index";
```

not `"dashboard"`.

---

The URL is determined by your mappings:

```java
@Controller
@RequestMapping("/dashboard")
public class DashboardController {

    @GetMapping
    public String index() {
        return "dashboard/index";
    }

}
```

Now visiting

```
localhost:8080/dashboard
```

renders

```
templates/dashboard/index.html
```

---

Similarly,

```java
@GetMapping("/settings")
public String settings() {
    return "dashboard/settings";
}
```

maps

```
GET /dashboard/settings
```

to

```
templates/dashboard/settings.html
```
